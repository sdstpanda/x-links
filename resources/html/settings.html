Filtering Click here to toggle the guide
Lines starting with / will be treated as regular expressions. (This is very similar to 4chan-x style filtering)
Lines starting with # are comments and will be ignored.
Lines starting with neither # nor / will be treated as a case-insensitive string to match anywhere.
For example, /touhou/i will highlight entries containing the string `touhou`, case-insensitive.

The lower a filter appears in this list, the greater its priority will be.

You can use these additional settings with each regular expression, separating them with semicolons:
  • Apply the filter to different scopes:
    tags;, title; or uploader;. By default the scope is title;tags;
  • Force a gallery to not be highlighted: If omitted, the gallery will be highlighted as normal
    bad:no;, bad:yes;, or just bad;
  • Only apply the filter to certain categories.
    only:doujinshi,manga;.
    Categories: artistcg, asianporn, cosplay, doujinshi, gamecg, imageset, manga, misc, non-h, private, western
  • Only apply the filter if it is not a certain category:
    not:western,non-h;.
  • Apply a colored decoration to the matched text:
    color:red;, underline:#0080f0;, or background:rgba(0,255,0,0.5);
  • Apply a colored decoration to the [Ex] or [EH] tag:
    link-color:blue;, link-underline:#bf48b5;, or link-background:rgba(220,200,20,0.5);.
  • Apply a colored decoration to BOTH the matched text and tag:
    colors:blue;, underlines:#bf48b5;, or backgrounds:rgba(220,200,20,0.5);.
Additionally, some settings have aliases. If multiple are used, only the main one will be used.
  • tags: tag
  • only: category, cat
  • not: no
  • colors: cs
  • underlines: us
  • backgrounds: bgs
  • color: c
  • underline: u
  • background: bg
  • link-color: link-c, lc
  • link-underline: link-u, lu
  • link-background: link-bg, lbg
